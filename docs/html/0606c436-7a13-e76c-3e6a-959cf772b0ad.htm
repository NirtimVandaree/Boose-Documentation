      BooseApp Namespace

      The BooseApp namespace contains application-specific replacements and helpers used by the BOOSE interpreter for coursework. It provides drawing canvas implementations, command replacements (moveto, drawto, pen, circle, etc.), and utility classes used by unit tests and the interpreter frontend.

      Classes

      AppArray Command implementation for creating runtime arrays via the 'array' keyword. Supports optional element type declaration (int/real) and validates name/size.
      AppArrayManager Simple runtime manager for arrays declared by `array` commands. Stores arrays by name with element type and size. Provides poke/peek helpers.
      AppCall 
      AppCanvas Represents a drawing canvas that implements the ICanvas interface. Responsible for handling drawing operations such as shapes, lines, colours, and text.
      AppCircle This class handles the CIRCLE command. It draws a circle on the canvas using one parameter: the radius.
      AppDrawTo This class handles the DRAWTO command. It draws a straight line from the current position to a new (x, y) point.
      AppElse Minimal permissive Else command replacement to avoid BOOSE compound-limit restriction. It is used to skip the else block when executed.
      AppEnd Minimal permissive End command replacement to avoid BOOSE compound-limit restriction. Provides handling for 'end while' and 'end for' to implement loop jumps.
      AppFor A permissive 'For' command replacement that supports simple numeric loops like: for i =1 to10 step1
      AppIf A permissive 'If' command replacement that accepts simple boolean expressions and allows nested commands between 'if'...'else'...'end if'.
      AppInt App-level replacement for the BOOSE "int" declaration command. Does NOT inherit from BOOSE.Int to avoid internal restriction checks. Manually registers a BOOSE.Int variable into the StoredProgram to satisfy parsing.
      AppMoveto This class handles the MOVETO command. It moves the drawing cursor to a new (x, y) position without drawing a line.
      AppPeek 
      AppPen Handles the PEN command which sets the drawing color using three integer parameters (r,g,b).
      AppPoke 
      AppReal App-level replacement for the BOOSE "real" declaration command. Does NOT inherit from BOOSE.Real to avoid internal restriction checks. Manually registers a BOOSE.Real variable into the StoredProgram.
      AppRect Represents a command that draws a rectangle on the canvas. This command accepts exactly two integer parameters: width and height.
      AppShared 
      AppStoredProgram StoredProgram replacement that removes built-in execution limits and provides debug tracing.
      AppWhile A permissive 'While' command replacement that accepts simple boolean expressions and allows nested commands between 'while'...'end while'.
      AppWrite This class handles the WRITE command. It writes text or numeric values to the canvas at the current cursor position.

      Interfaces

      IAppCanvas Application-specific canvas contract used by the App code and unit tests. Declares a minimal set of operations provided by AppCanvas used by the app.
      IAppStoredProgram Minimal application-level abstraction for StoredProgram used by the app code. Exposes members that AppStoredProgram already provides and that other app code may rely on. Method names avoid clashing with base class signatures by appending 'Obj'.
